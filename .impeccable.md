## Design Context

### Users
Privacy-conscious people who hate ad-bloat on traditional weather sites.
International audience (15 languages). Mostly mobile + desktop browser usage.
Found the site via tech communities (r/InternetIsBeautiful), Ko-fi tippers, and word of mouth. They notice when something is fast and when it isn't.

### Brand Personality
- **Honest** — no tracking, no cookies, no ads, no dark patterns. Says what it does.
- **Fast** — 530KB total. Loads in a blink. Respects user time.
- **Quietly opinionated** — the brand name itself is a stance. It's not neutral; it's against something.

The interface should feel like a tool made by someone who actually cares,
for users who actually care. No corporate gloss, no growth-hacking
patterns, no engagement bait.

### Aesthetic Direction
Currently the site uses a refined-utilitarian default look: system font stack, blue accent (`#2563eb` light / `#3b82f6` dark), card sections with 8-12px radius, 16px-base spacing.

Dan wants to ADD alternate visual styles users can opt into — same
layout structure (same sections, drag/resize/close), but different
fonts / colors / corner radii / spacing rhythm. To be implemented
one at a time:

1. **Editorial / magazine** — refined serif display, generous whitespace,
   restrained accent. Feels like the Sunday weather section of a
   considered publication.
2. **Retro-meteorological** — chart-paper / cream tones, vintage
   forecast-bulletin feel. Slightly nerdy, slightly nostalgic.
3. **Refined minimalism** — quiet, precise, single accent, breath.
   Apple-grade attention to spacing and type — but distinct from Apple.

The default look stays available for users who don't want to switch.

### Constraints / Anti-references
- No frameworks, vanilla JS only
- Static GitHub Pages — no build step
- Must work on mobile first (a lot of traffic is phone)
- Dan is colorblind — high-contrast palettes required
- Avoid AI-slop tells: gradient text, glassmorphism walls, side-stripe
  card borders, generic icon-above-heading templates, monoculture
  fonts (no Inter / DM Sans / Plus Jakarta / Fraunces / Crimson /
  Playfair / Cormorant — these are reflex defaults to reject)
- Light AND dark modes must work for every alternate style

### Design Principles
1. **Layout is sacred** — alternate styles change look, never structure.
   Sections stay where they are; drag/resize/close still works.
2. **Theme axis × Style axis** — light/dark and visual style are
   independent. Every style works in both modes.
3. **Typography drives identity** — most of the differentiation across
   styles comes from font choice + spacing rhythm, not from accent
   color shuffling.
4. **Performance is part of the brand** — no web-fonts that bloat the
   load. Use `font-display: swap`, subset to Latin where possible,
   keep total fonts ≤ 2 per style.
5. **High-contrast is non-negotiable** — every palette passes WCAG AA
   for body text against background, in both light and dark modes.
